# plans/

Planning-session notes, one markdown file per session.

## Naming convention
`YYYY-MM-DD-<short-topic>.md`

Examples:
- `2026-07-08-data-sources.md`
- `2026-07-10-puhekieli-eval.md`

## What goes in a plan file
- **Date** and **topic** (in the filename and an H1).
- **Goal** — what we're trying to decide/build.
- **Context / constraints** — relevant facts, links to `.agents/` docs.
- **Open questions** — unknowns blocking the plan (mark clearly).
- **Decision / approach** — once settled.
- **Next actions** — concrete steps.

Plans are the "thinking"; `.agents/DECISIONS.md` is the short append-only record of
*what* we decided. When a plan resolves a decision, copy a 2-line summary there.
